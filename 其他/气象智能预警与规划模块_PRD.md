## 1. 项目背景与目标

### 1.1 项目背景

IWN 廊道运维管理平台现阶段已具备基础的廊道信息管理、飞行器管理、节点管理等能力。随着业务规模扩大，无人机在廊道内的飞行安全面临来自气象环境的威胁日益升级——强风、强降水、低能见度等恶劣气象条件是导致飞行事故的重要诱因。

任务书明确要求，基于对接外部专业气象服务商的网格化气象数据，建立完整的"**气象数据接入 → 禁飞规则计算 → 禁飞区动态生成 → 任务创建拦截 → 柔性重规划**"全链路气象智能预警与规划体系。

### 1.2 核心目标

| 目标 | 描述 |
|------|------|
| **数据接入** | 通过 API 对接专业气象服务平台，按分钟/小时/日三频率拉取高、中、低精度网格气象数据 |
| **规则引擎** | 基于不同机型大类（多旋翼/VTOL/固定翼）的气象耐受极限，动态生成三级气象禁飞区 |
| **透明化监控** | 为管理员提供数据接入链路健康监控仪表盘，具备异常告警与手动重试能力 |
| **任务安全校验** | 在任务创建入口前置气象冲突检测，自动拦截高风险飞行任务 |
| **智能降级方案** | 对被拦截任务提供"顺延时间窗口"和"空间绕行航线"两大智能重规划方案 |

### 1.3 技术栈与图注

- **前端框架：** Vue 3 + TypeScript + Vite
- **UI 组件库：** Element Plus 2.7
- **图表/地图：** ECharts 5、Leaflet 1.9、Canvas 2D API
- **样式方案：** Tailwind CSS + Scoped CSS
- **路由管理：** Vue Router 4，所属一级导航「廊道基础数据」及「规划计划」

## 2. 系统整体架构

### 2.1 功能模块拓扑图

```
IWN 廊道运维管理平台

│   ├── 廊道信息维护

│   ├── ★ 气象禁飞区数据池         → WeatherNfzDataPool.vue

│   ├── 空域与航路维护
│   └── 受限区域总览
│
└── 规划计划 (planning)
    └── ★ 用户任务信息管理（改造）  → UserTask.vue
        ├── 系统预检状态列
        ├── 失败原因筛选
        └── 任务详情/核验抽屉

            ├── 3D 航线态势图（Canvas）
            └── 智能重规划操作区
```

### 2.2 气象数据链路全貌

```
外部气象服务商 API
        │
        ▼  (CRON 定时拉取: 10min / 1h / 2h)
气象数据接入层 (WeatherDataSource)
  - 高精度实况网格 (1km×1km，1次/分钟)
  - 中精度短期预报 (5km×5km，1次/15分钟，未来24h)
  - 低精度延伸预报 (20km×20km，1次/2h，未来15天)
        │
        ▼  解析 / 映射 / 入库
气象网格数据池
  字段: u10, v10 → 风速风向
        t2m     → 地面温度
        prate   → 降水率
        vis     → 能见度
        │
        ▼  规则引擎触发 (WeatherRulesConfig 中配置)
三级禁飞区计算引擎
  一级绝对禁飞区 (RED)   ← 固定翼阈值被突破
  二级禁飞区 (ORANGE)    ← VTOL 阈值被突破
  三级禁飞区 (YELLOW)    ← 多旋翼阈值被突破
        │
        ▼  写入禁飞区数据池 (WeatherNfzDataPool)
动态禁飞区列表
  - 网格编号、禁飞级别、限制机型、生效时段、触发诱因、状态
        │
        ▼  任务创建前校验 (UserTask / TaskDetail)
飞行任务预检引擎
  - 拦截 → 展示安全报告 → 禁用"审核通过"按钮
  - 通过 → 常规审核流程
        │
        ▼  智能重规划
  📅 时间顺延：推荐24h内适飞时间窗口
  🗺️ 空间绕行：生成绕行备选路径
```

## 3. 模块一：气象数据源监控与调度管理

**文件路径：** `src/views/corridor/WeatherDataSource.vue`  
**路由路径：** `/corridor/weather-data-source`  
**侧边栏菜单：** 廊道基础数据 → 数据源调度监控

### 3.1 页面定位

本页是气象数据接入全链路的"监控室"与"操作台"，供系统管理员：
1. 实时感知外部气象 API 的连接健康状态
2. 配置 API 接入参数与调度频率
3. 查看并追溯每次数据拉取的运行日志

### 3.2 页面布局

```
┌─────────────────────────────────────────────────────────┐
│  [服务状态] [最后同步] [有效网格数] [调用次数+进度条]      │  ← 顶部 4 个健康指标卡
├──────────────────┬──────────────────────────────────────┤
│                  │                                      │
│  API配置表单     │   近期拉取与解析日志 (Top 50)          │
│  (col: 10/24)    │   (col: 14/24)                       │
│                  │                                      │
│  · 服务商名称    │   [全部] [成功] [失败/异常] 快速筛选   │
│  · Endpoint(遮蔽)│   ┌────────────────────────────────┐│
│  · 拉取频率      │   │调度时间│耗时│数据量│状态&详情│操作│ │
│  · BBox范围      │   ├────────────────────────────────┤│
│  · 要素映射状态  │   │  数据行...                      ││
│                  │   └────────────────────────────────┘│
│  [保存配置]      │                                      │
└──────────────────┴──────────────────────────────────────┘
```

### 3.3 功能需求详解

#### 3.3.1 顶部健康指标卡 (4列)

| 卡片 | 内容 | 交互/状态 |
|------|------|-----------|
| 服务连接状态 | 🟢 在线/正常运转 或 🔴 离线/连接异常 | 绿点呼吸动画（在线）/静止红点（离线）|
| 最后同步时间 | 精确到秒的时间戳 + "距今已过 N 分钟" | 超过配置间隔 2 倍时文字变橙色预警 |
| 当前有效网格数 | 数字（如 1,240 个）| 静态展示 |
| 今日接口调用次数 | 已用/配额（如 144/500 次）+ 进度条 | 使用率 >80% 时进度条变红 |

#### 3.3.2 左侧 API 配置表单

**字段定义：**

| 字段 | 类型 | 说明 |
|------|------|------|
| 服务商名称 | 只读文本 Input | 如"省级专业气象服务平台" |
| API Endpoint | 密码型 Input（可切换显示）| 保护敏感请求地址 |
| 拉取频率 | Select 下拉 | 选项：每 10 分钟 / 每 30 分钟 / 每小时 |
| 请求空间范围 (BBox) | 只读 Input（Monospace 字体）| 如 `Lat: 28.5~31.2, Lon: 118.0~123.0` |

**核心气象要素映射区（Check List 展示）：**

该区块以浅蓝色背景卡片展示，对每项解析映射关系进行状态确认：

| 状态标识 | 要素 | 说明 |
|----------|------|------|
| ✅ 已映射解析 | 风速风向 | 由 u10, v10 向量合成计算 |
| ✅ 已映射解析 | 地面温度 | 字段 t2m |
| ✅ 已映射解析 | 降水率 | 字段 prate |
| ✅ 已映射解析 | 能见度 | 字段 vis |

*注：若字段未映射或解析失败，应显示 ❌ 状态，标注具体错误。*

**底部操作：** `[保存配置更新]` 主按钮，修改后持久化写入配置。

#### 3.3.3 右侧同步日志表格

**顶部筛选（Radio Button Group，行内）：** `全部记录` / `成功` / `失败/异常`

**表格列定义：**

| 列名 | 宽度 | 说明 |
|------|------|------|
| 调度时间 | 100px | 精确到秒（Monospace 字体） |
| 任务耗时 | 90px | 如 1.2s |
| 获取数据量 | 100px | 如 4.2 MB / 0 Bytes |
| 状态详情 | 自适应 | 图标 + 状态类型 + 详情文案（双行展示）|
| 操作 | 160px | [查看详情] / [手动重试] |

**状态类型：**
- 🟢 **成功** — 解析入库完毕，已更新多级禁飞区
- 🔴 **失败** — 接口超时/无响应
- 🟡 **异常** — 数据格式校验不通过（如风速负值）

**[查看详情] 操作：** 点击后右侧滑出 `el-drawer`，内含深色主题（`#1e1e1e`）的 Raw JSON 代码预览区，供开发人员 Debug 使用。

**[手动重试] 操作：** 仅对失败/异常记录展示，点击后触发单次重新拉取。

└──────────────────────────────────────────────────────────┘

┌─────────────────┐ ┌─────────────────┐ ┌─────────────────┐
│ 🟡 多旋翼无人机 │ │ 🟠 垂直起降/复合翼│ │ 🔴 纯固定翼无人机│
│ (Multi-rotor)   │ │ (VTOL)           │ │ (Fixed-wing)    │
│                 │ │                 │ │                 │

│                 │ │                 │ │                 │

└─────────────────┘ └─────────────────┘ └─────────────────┘
         1列                  2列                  3列

### 4.3 顶部配置警示区

- **组件：** `el-alert`，`type="warning"`，不可关闭
- **文案：** "配置提醒：修改以下机型类别的气象阈值后，系统将结合最新气象网格数据，自动重算并生成未来24小时的【多级气象禁飞区】。请谨慎操作。"

### 4.4 三张配置卡片（核心配置区）

#### 4.4.1 卡片通用结构

```
卡片 Header (彩色背景)
  ├── 机型图标（圆形 Icon 区）
  ├── 机型主标题 + 英文副标题
  └── 影响范围标签 (el-tag，颜色对应级别)

  └── el-form (label-position="left", label-width="120px")

      └── 工作温度      → el-slider (range 双端, 后缀: ℃)

  ├── [修改日志] 文字链接按钮 (次级)

#### 4.4.2 三卡片差异对照表

```

### 5.3 顶部统计卡片

| 卡片 | 数值颜色 | 背景 | 图标 |
|------|---------|------|------|
| 当前生效中 一级禁飞区 | text-red-600 | from-red-50 to-white | ri-alert-fill |
| 当前生效中 二级禁飞区 | text-orange-500 | from-orange-50 to-white | ri-error-warning-fill |

*注：数值应由后端实时接口提供，前端展示时以大号加粗字体突出。*

### 5.4 搜索筛选栏

| 筛选项 | 组件类型 | 说明 |
|--------|---------|------|
| 生效时段 | DatetimePicker（范围，精确到小时）| 筛选时间段内有效的记录 |
| 禁飞级别 | Select 下拉 | 选项：全部 / 一级绝对禁飞 / 二级禁飞 / 三级禁飞 |
| 空间范围 | Input 文本框 | 提示：请输入网格编号或廊道ID |
| 当前状态 | Radio Button Group | 全部 / 即将生效 / 生效中 / 已失效 |

### 5.5 核心数据表格

**表格属性：** 斑马纹 (`stripe`)、边框 (`border`)、高度填满父容器 (`height="100%"`)

**列定义：**

| 列名 | 宽度 | 渲染方式 |
|------|------|---------|
| 网格/区域编号 | 160px | Monospace 字体文本（如 Grid-A02-15） |
| 禁飞级别 | 140px | 彩色 Tag：🔴一级、🟠二级、🟡三级 |
| 限制进入机型 | 200px | 灰色小 Tag 组合（如 [全机型禁入] / [多旋翼禁入] [VTOL禁入]）|
| 生效时间段 | 280px | 时钟图标 + "开始时间 至 结束时间" |
| 系统触发诱因 | 自适应 | 长文本省略显示，完整内容在 Tooltip 中 |
| 状态 | 120px | 状态圆点（动态光晕）+ 文字：生效中 🟢 / 即将生效 🔵 / 已失效 ⚪ |
| 操作 | 200px (fixed right) | [空间预览] / [人工解除]（仅对未失效记录显示） |

**[人工解除] 操作流程：**
1. 点击后弹出 `el-message-box` 二次确认
2. 确认文案：「强制解除后，网格 [XXX] 的飞行限制将失效。该操作会在系统中留下高危日志，确认继续？」
3. 确认按钮："强制解除"（危险红色）

### 5.6 分页器

采用精简线框风格，与项目其他页面保持一致：

### 7.5 三套气象渲染引擎

#### 7.5.1 引擎一：IDW 温度/降水 Canvas 云图

**技术：** 自定义 `WeatherCanvasLayer`（扩展 `L.Layer`），使用 Canvas 2D API + 反距离权重插值（IDW）。

**渲染流程：**

```
1. 按 timeOffset 过滤网格数据
2. LatLng → Canvas 像素坐标转换
3. 创建 160×120 离屏 Canvas（降采样加速）
4. 逐像素 IDW 插值：weight = 1/(dist² + ε)，加权均值
5. 色彩映射（RGBA）：
   · 温度：norm=(temp-10)/25，蓝(冷)→绿(中)→红(热)
   · 降水：蓝色通道透明度随 rain 值线性加深
   · 双叠：rain 增强蓝通道、削弱红通道，协同融合
6. 离屏 Canvas 放大（高质量双线性平滑）至全图尺寸
7. globalAlpha=0.8 半透明叠加，不遮挡底图
```

**地图联动：** 监听 `moveend / viewreset / resize / zoomend`，触发 `_update()` 重新投影所有网格坐标，图层与地图视角严格同步，不会错位。

#### 7.5.2 引擎二：粒子流体风场动画

**技术：** 独立 Canvas（`windCanvasRef`，`z-index: 401`），RAF 循环驱动，`destination-out` 拖尾消散。

| 参数 | 值 |
|------|----|
| 粒子数 | 350 个，全图随机均布 |
| 生命周期 | 60 ~ 120 帧，超出后原地重生 |
| 位移 | `windSpeed × 0.5` px/帧（IDW 加权该像素点风速）|
| 拖尾 | `rgba(0,0,0,0.20)` destination-out 擦除 |
| 色彩 | 低速→青蓝，高速(≥12m/s)→暖黄白 |

- `showWind=true` → `startWindAnimation()`（resize + spawnParticles + requestAnimationFrame 循环）
- `showWind=false` → `stopWindAnimation()`（cancelAnimationFrame + clearRect 清空）

#### 7.5.3 引擎三：气象禁飞区 Leaflet Polygon

**技术：** `L.LayerGroup`（`nfzLayerGroup`），每次状态变化先 `clearLayers()` 再按时间切片筛选重绘。

**时间切片过滤：**
```typescript
nfz.effectiveTimeRange[0] <= timeOffset <= nfz.effectiveTimeRange[1]
```

| 级别 | fillColor | 典型场景 |
|------|-----------|---------|
| RED | `#ff4d4f` | 一级：极端风速/雷暴，全机型禁入 |
| ORANGE | `#fa8c16` | 二级：中度风雨，多旋翼/VTOL禁入 |
| YELLOW | `#fadb14` | 三级：轻度风雨，微型旋翼机禁入 |

**Tooltip（sticky，跟随鼠标）：** 展示区域名称、管控时段、受禁机型、警戒原因，均有对应颜色标注。

### 7.6 Watch 联动机制

```typescript
watch(
  () => [showTemp, showRain, showWind, showNoFlyZone, timeOffset],
  () => updateWeatherCanvas(),
  { deep: true }
)

function updateWeatherCanvas() {
  weatherCanvasInstance._update()          // 重绘 IDW 云图
  showWind ? startWindAnimation() : stopWindAnimation()
  nfzLayerGroup.clearLayers()
  if (showNoFlyZone) renderNfzPolygons()  // 重绘禁飞区
}
```

## 10. 接口对接要求

### 10.1 外部气象 API 接口规格

根据任务书要求，外部气象服务商应提供以下三类数据接口：

| 接口类型 | 分辨率 | 拉取频率 | 预报时效 |
|---------|--------|---------|---------|
| 高频实况网格 | 1km × 1km | 1次/分钟 | 实时 |
| 短期预报网格 | 5km × 5km | 1次/15分钟 | 未来 24h |
| 延伸预报网格 | 20km × 20km | 1次/2小时 | 未来 15天 |

**核心要素映射：**

| 提供字段 | 平台字段 | 说明 |
|---------|---------|------|
| u10, v10 | windSpeed + windDirection | 矢量合成 |
| t2m | temperature | 地面 2m 温度 |
| prate | precipitation | 降水率 (mm/h) |
| vis | visibility | 能见度 (m) |

### 10.2 内部接口规格（前后端联调）

| 接口名称 | 方法 | URL | 说明 |
|---------|------|-----|------|
| 获取数据源健康状态 | GET | `/api/weather/source/status` | 返回连接状态、最后同步时间、网格数、今日调用量 |
| 获取调度日志列表 | GET | `/api/weather/sync-logs?limit=50` | 返回近50条拉取日志 |
| 手动触发一次拉取 | POST | `/api/weather/sync/trigger` | 手动重试单次拉取 |
| 获取规则配置 | GET | `/api/weather/rules` | 返回三机型阈值配置 |
| 保存规则配置 | PUT | `/api/weather/rules/{type}` | 保存单机型阈值，触发后端重算禁飞区 |
| 获取禁飞区列表 | GET | `/api/weather/nfz?page=1&size=20` | 带分页、过滤参数 |
| 人工解除禁飞区 | POST | `/api/weather/nfz/{id}/release` | 记录高危审计日志 |
| 任务气象预检 | POST | `/api/task/precheck` | 入参：机型、时间段、航线坐标；出参：预检结果、冲突信息 |
| 获取适飞时间窗口 | POST | `/api/weather/forecast/slots` | 入参：机型、航线；出参：未来24h各时段适飞状态 |
| 获取态势图网格数据 | GET | `/api/weather/grids?offset={h}` | 入参：时间偏移(0-24)；出参：对应时刻全域网格数据集合 |
| 获取态势图禁飞区（实时）| GET | `/api/weather/nfz/active?offset={h}` | 入参：时间偏移；出参：当前切片生效的禁飞区多边形集合 |
